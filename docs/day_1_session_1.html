Abstract

We begin with an overview of Python’s object model, covering the concepts of objects, name binding, namespaces, and garbage collection. We then work through Python’s scalar types: integers, floating-point numbers, Booleans, and strings. For each type we examine how objects are created and bound to names, what operators apply to them, and which built-in functions are available. For strings, we also cover f-string formatting, indexing and slicing, and string methods. The session closes with an overview of type conversion between the scalar types.

Python’s Object Model

Everything in Python is an object. An object is a structure held in memory with three properties:

  • It has an identity, which is a unique identifier corresponding to its location in memory, accessible via the built-in function id().
  • It has a type, which determines what kind of thing it is and what operations are valid on it, accessible via type().
  • It has a value, which is the actual data it contains.

When you write x = 5 in Python, you create an integer object with value 5, and the name x is then bound to that object. The = operator in Python is a binding operator. The right-hand side is evaluated first, producing an object, and the left-hand side is a name that is then made to refer to that object.

This distinction between binding and storing matters in practice. A name is not an object. It is a reference to an object. Multiple names can refer to the same object simultaneously, and a name can be rebound to a different object at any time.

Names are stored in namespaces. A namespace is a lookup table mapping names to the objects they refer to. When you bind a name at the top level of a script or module, it goes into the global namespace. Function calls each create their own local namespace. Looking up a name means searching the relevant namespace, or chain of namespaces, for a matching entry.

An object remains in memory as long as at least one name or other reference points to it. When nothing refers to an object, Python’s garbage collector reclaims that memory automatically. You can explicitly remove a name from its namespace with del x. If x was the only reference to the object it pointed to, that object then becomes eligible for collection.

This object model applies uniformly to every Python object without exception. Python’s built-in types divide broadly into two categories. There are types that hold a single value, and there are collection types that hold multiple values. The former — sometimes called scalar types, or elementary types — are the subject of this document. Specifically, we cover integers, floating-point numbers, booleans, and strings. Collection types such as lists, tuples, and dictionaries are covered in a later session.

Integers

Python has two commonly used numeric types: integers and floats. Integers (int) represent whole numbers, and floats (float) represent numbers with a decimal part. We start with integers.

The three lines below bind the names x, y, and z to integer objects.

x = 5
y = 10
z = -3

Recall what is happening here. Each line creates an integer object and binds a name to it. No output is produced because these are binding operations, not expressions. The objects now exist in memory and the names refer to them.

To see the value of x, we evaluate it as an expression.

x
5

We can inspect the object more directly. id() returns the identity of an object, a unique integer corresponding to its location in memory for the duration of the current session.

id(x)
140533294213616

The specific number is not meaningful and will differ between runs. It is simply a way of confirming that this is a distinct object with a unique memory address.

type() returns the type of the object.

type(x)
int

The output int confirms that x refers to an integer.

Note that in Python, types and classes are the same thing. The type int is a class, and x is an instance of it. The built-in function isinstance() makes this explicit.

isinstance(x, int)
True

It returns True, confirming that x is an instance of int. The return value is a Boolean, a type we will cover shortly.

It is worth pausing here to look more carefully at how binding and rebinding work, as this is a general principle that applies to all Python objects, not just integers. Consider the following sequence.

a = 5
b = a
a = 3

a = 5 creates an integer object with value 5 and binds the name a to it. b = a does not copy anything. It simply binds the name b to the same object that a already refers to. a = 3 creates a new integer object with value 3 and rebinds a to it. At this point b still refers to the original object with value 5, unaffected. No copying took place. Only names were bound or rebound.

In general, objects are things we can act on and do things with. In Python there are two general mechanisms for acting on objects: operations, performed using operators, and functions. Both are introduced here in the context of integers, though both apply equally to all Python types.

An operation is an action applied to one or more objects to produce a result. An operator is a symbol that denotes a particular operation. The + in x + y is an operator; the addition it performs is the operation. The values an operator acts on are called its operands. Most operators take two operands, one on each side, and are called binary operators. A smaller number take a single operand and are called unary operators. The minus sign in -3 is a unary operator; the plus sign in x + y is a binary operator.

What an operator does depends on the types of its operands. The + applied to two integers performs addition. On the other hand, as we will see shortly, the same + applied to two strings performs concatenation, joining them end to end. This behaviour, where one operator symbol performs different operations depending on the types of its operands, is called operator overloading.

The following code shows the standard arithmetic operators applied to the integer variables defined above:

x + y  # addition
y - x  # subtraction
x * y  # multiplication
y / x  # division; returns a float
y // x  # integer division
y % x  # modulo (remainder after division)
x**2  # exponentiation

Note that each of these lines is an expression. An expression is a piece of code that evaluates to a value. This is distinct from a statement, which is a complete executable instruction. Assignment is a statement: x = 5 performs a binding but does not itself produce a value. An expression written on its own line forms an expression statement, which Python evaluates and, in an interactive session, displays the result of. Like any value, the result of an expression can also be bound to a name: z = x + y binds the result of the addition to the name z.

Another note for those coming from languages such as R or MATLAB: Python uses ** for exponentiation, not ^. The caret has a different meaning in Python and is not used for powers.

A further note is that all of these integer operations just mentioned return an integer when both operands are integers, with one exception. Division with / always returns a float, even when the result is a whole number. So 10 / 5 returns 2.0, not 2. Integer division with // discards the remainder and returns an integer, so 10 // 3 returns 3.

Beyond operators, Python provides functions. A function is a named, callable object that takes zero or more input objects, called arguments, and returns a new object. We will look at functions in depth shortly. For now it is enough to know that a small number of functions are available as part of the Python language itself. These are called built-in functions. There are around 70 of them in total, a deliberately small and stable set.

The following built-in functions work with integers.

abs(z)  # absolute value
pow(x, 2)  # raise x to the power 2
max(x, y)  # larger of the two values
min(x, y, z)  # smallest of the three values

Like operators, functions behave differently depending on the types of their arguments. What max() does when given integers differs from what it does when given strings. The same function name, applied to different types, performs a correspondingly appropriate operation. This is the same principle as operator overloading, applied to functions.

We just mentioned that Python has around 70 built-in functions. However, across all of Python’s vast ecosystem of packages, there are probably more like millions of available functions. This is a useful point at which to note the layered structure of Python. The innermost layer is the Python language itself: its syntax, its operators, and its built-in functions. Surrounding that is the Python standard library, which is a large collection of modules that are officially maintained with the language itself and always ship with Python. These packages can be imported when needed, as we will see. Beyond that is the vast ecosystem of third-party packages.

Floats

A float is a number with a decimal part. Python’s float type stores numbers in 64-bit double-precision floating-point format, which provides approximately 15 to 17 significant decimal digits of precision.

In the examples below, we rebind the names x, y, and z, which previously referred to integers, to float objects:

x = 3.14
y = 2.718
z = -1.5

Note that unlike in some languages, no type declaration is required and no error is raised when we rebind from one type to another. Python does not require names to be associated with a fixed type in advance. A name can be rebound to an object of any type at any time. This property is called dynamic typing, and it distinguishes Python from statically typed languages such as C or Java, where a variable’s type must be declared before use.

The arithmetic operators work with floats in the same way as with integers.

x + y    # addition
y - x    # subtraction
x * y    # multiplication
y / x    # division
x ** 2   # exponentiation

Likewise, functions can be used with floats. For example, two built-in functions commonly used with floats are abs(), which returns the absolute value, and round(), which rounds to a specified number of decimal places.

abs(z)       # absolute value: 1.5
round(x, 1)  # round to 1 decimal place: 3.1

Python supports scientific notation for writing very large or very small numbers. The letter e is read as “times ten to the power of.” So 1e6 means one million (1 times 10 to the power of 6), and 2.5e-3 means 0.0025 (2.5 times 10 to the power of -3). Numbers written in this notation are floats. In other words, 1e2, which is 100, is in fact represented as the float 100.0.

1e6      # 1000000.0
2.5e-3   # 0.0025

Python also permits underscores within numeric literals as a visual separator. The underscore has no effect on the value and is there solely to improve readability for large numbers.

1_000_000   # same as 1000000
0.000_001   # same as 0.000001

This underscore syntax was introduced in Python 3.6.

Booleans

A Boolean is a logical variable: a variable that can be logically true or false. In Python, a Boolean object is an object that can take exactly one of two values: True or False. These are the only two possible values for this type. True and False are reserved keywords in Python. They are literal values, in the same way that 5 is a literal integer value or 3.14 is a literal float value. In other words, in Python, True is not the same as "True". The former is a Boolean object; the latter is a string object that happens to contain the characters T, r, u, e.

We bind Boolean values to names just as we do with any object in Python.

is_valid = True
is_empty = False

This is a good point to say something about naming. So far, names have been short and anonymous: x, y, z, a, b. For the Boolean examples just given, we could equally have written x = True and y = False. The names is_valid and is_empty are simply more informative. In general in Python, names can be anything that begins with a letter or an underscore. It can contain letters, digits, and underscores. It cannot be a Python keyword. Names are case-sensitive: value and Value are different names. Beyond what is merely valid, there is a widely followed set of conventions documented in PEP 8, the official Python style guide. PEP 8 recommends lowercase names with underscores separating words, a convention known as snake_case. It also recommends choosing names that are descriptive and readable. For Boolean values, names that read as a condition or question tend to work well.

The type and class of these Boolean objects is bool.

type(is_valid)
bool
isinstance(is_valid, bool)
True

The primary operators for Booleans are the logical operators and, or, and not.

is_valid and is_empty   # True only if both operands are True
is_valid or is_empty    # True if at least one operand is True
not is_valid            # negates the value

Comparison operators also produce Boolean values. They take two operands and evaluate to either True or False.

5 > 3    # greater than
5 < 3    # less than
5 >= 5   # greater than or equal to
5 == 5   # equal to
5 != 3   # not equal to

These equality/inequality operators work with integers, floats, strings, and many other types. What they means in practice may depend on the types involved.

Some of the built-in functions that work with numbers also accept Boolean arguments.

min(True, False)
False

Strings

A string is a sequence of characters used to represent text. String objects can be created using single quotes, double quotes, or triple quotes for multi-line strings.

s1 = 'Hello'
s2 = "World"
s3 = '''This is a
multi-line string'''

The choice between single and double quotes is a matter of style. Triple quotes allow the string to span multiple lines.

Several operators can be applied to strings. The most commonly used are +, which concatenates two strings, *, which repeats a string a given number of times, and in, which tests whether one string appears within another.

s1 + ' ' + s2
'Hello World'
'Ha' * 3
'HaHaHa'
'Hello' in s2
False
'Hello' not in s2
True

A number of built-in functions also work with strings, the most commonly used of which is len(), which returns the number of characters.

len(s1)
5

As we will see shortly, strings also have a very large number of functions built directly into the object itself. Functions built into an object are called methods, and they are among the most important tools in Python.

Python’s f-strings provide a concise way of embedding expressions inside strings. An f-string is prefixed with f and uses curly braces to mark expressions that are evaluated and inserted when the string is created.

name = 'Alice'
age = 30
f"My name is {name} and I am {age} years old"
'My name is Alice and I am 30 years old'

Expressions inside the braces are evaluated at the point where the f-string is defined.

f"In five years I will be {age + 5}"
'In five years I will be 35'

F-strings also support format specifications for controlling how values are displayed.

f"Pi to 3 decimals: {3.14159:.3f}"
'Pi to 3 decimals: 3.142'
f"Large number: {1_000_000:,}"
'Large number: 1,000,000'
f"Percentage: {0.856:.1%}"
'Percentage: 85.6%'

An older but still widely used alternative f-strings is the .format() method.

"Hello, {}!".format("World")
'Hello, World!'
"{} + {} = {}".format(2, 3, 5)
'2 + 3 = 5'
"{name} is {age}".format(name="Bob", age=25)
'Bob is 25'

Because a string is a sequence of characters, individual characters can be accessed by position using square brackets. This is known as indexing (or slicing) and is not unique to strings. It applies to any Python sequence type, and we will encounter it again when we cover lists and tuples.

When indexing or slicing in Python, positions are numbered from zero:

text = "Hello, World!"
text[0] # first character
'H'
text[1] # second character
'e'

Negative indices count from the end.

text[-1] # last character
'!'
text[-2] # second to last character
'd'

A range of characters can be extracted using a slice:

text[0:5]   # characters at positions 0 through 4
'Hello'

Strings have a large number of methods. A method is a function that belongs to a specific object and is called using dot notation. The call text.lower() means: apply the lower function that belongs to the string object text. This is functionally the same as a built-in function, but the function is attached to the object rather than being free-standing.

text.lower()                     # convert to lowercase
text.upper()                     # convert to uppercase
text.strip()                     # remove leading and trailing whitespace
text.replace("World", "Python")  # replace a substring
"a,b,c".split(",")               # split on a delimiter

There are many more string methods than shown here. The Python documentation provides the full list.

Python 3 strings are Unicode by default. They can represent characters from any writing system, not just the Latin alphabet.

"café"
"日本語"

When text must be stored or transmitted as raw bytes, it must be encoded. The most common encoding is UTF-8. Encoding converts a string to a bytes object; decoding converts it back.

"café".encode('utf-8')           # b'caf\xc3\xa9'
b'caf\xc3\xa9'.decode('utf-8')  # 'café'

Converting Between Types

Python provides built-in functions for converting objects from one type to another. The conversion function takes the same name as the target type.

int(3.9)          # float to int: truncates toward zero, gives 3
float(42)         # int to float: gives 42.0
str(42)           # int to string: gives '42'
bool(0)           # 0 converts to False
bool(1)           # any non-zero integer converts to True
bool("hello")     # non-empty string converts to True
bool("")          # empty string converts to False

Conversion is not always possible. Attempting to convert a non-numeric string to an integer raises an error rather than silently producing a wrong result.

int("hello")   # raises ValueError

Python raises an exception on invalid operations rather than guessing at intent. An explicit error is easier to diagnose than a silent incorrect result.
