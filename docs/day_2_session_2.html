Abstract

This session introduces Pandas, the primary library for data manipulation in Python. We begin with the two fundamental data structures, Series and DataFrame, and cover how to load data from CSV files. We then cover the core operations on DataFrames: selecting columns with square brackets, loc, and iloc; renaming columns; filtering rows with boolean conditions and query; adding and transforming columns with assign; sorting; and the split-apply-combine pattern using groupby and aggregation.

Series and DataFrames

Pandas provides two data structures built on top of NumPy arrays: Series for one-dimensional labelled data and DataFrame for two-dimensional tabular data. A Series is an array paired with an index.

import pandas as pd
import numpy as np

s = pd.Series([10, 20, 30], index=['a', 'b', 'c'])
s
a    10
b    20
c    30
dtype: int64

Arithmetic operations apply element-wise, just as with NumPy arrays.

s * 2
a    20
b    40
c    60
dtype: int64

A DataFrame is a table of columns, each a Series sharing a common index. The quickest way to create a small one by hand is from a dictionary of lists.

df = pd.DataFrame({'x': [1, 2, 3], 'y': [4, 5, 6]})
df
x y
0 1 4
1 2 5
2 3 6

In practice, data almost always comes from a file. pd.read_csv reads a CSV into a DataFrame and is the starting point for most real analyses. It accepts either a file path or a URL, which is convenient for publicly hosted data.

Groupby and Aggregation

groupby splits the DataFrame into groups by the values of one or more columns. An aggregation function applied to the result is computed separately within each group and the results are combined. This is the split-apply-combine pattern.

affairs.groupby('gender')['affairs'].mean()
gender
female    1.419048
male      1.496503
Name: affairs, dtype: float64
affairs.groupby('children')['affairs'].mean()
children
no     0.912281
yes    1.672093
Name: affairs, dtype: float64

agg applies multiple aggregation functions at once.

affairs.groupby('gender')['affairs'].agg(['mean', 'std', 'count'])

gender
female 1.419048 3.309264 315

Reshaping: Wide and Long

Data is often stored in wide format, with one row per subject and one column per measurement occasion or variable. Analysis and plotting frequently require long format, where each row is one observation. melt converts from wide to long; pivot converts from long back to wide.

Start with a small wide DataFrame: three students, each with a score in three subjects.

wide = pd.DataFrame({
    'name':    ['Alice', 'Bob', 'Charlie'],
    'maths':   [90, 75, 85],
    'english': [85, 80, 70],
    'science': [88, 92, 78]
})
wide
name maths english science
0 Alice 90 85 88
1 Bob 75 80 92
2 Charlie 85 70 78

melt takes the id_vars argument — the columns that identify each row and stay fixed — and folds every other column into two new columns: one for the variable name and one for the value.

long = wide.melt(id_vars='name', var_name='subject', value_name='score')
long
name subject score
0 Alice maths 90
1 Bob maths 75
2 Charlie maths 85
3 Alice english 85
4 Bob english 80
5 Charlie english 70
6 Alice science 88
7 Bob science 92
8 Charlie science 78

pivot reverses the operation. index specifies the column that becomes the row labels, columns specifies the column whose values become the new column names, and values specifies the column that fills the cells.

long.pivot(index='name', columns='subject', values='score')
subject english maths science
name
Alice 85 90 88
Bob 80 75 92
Charlie 70 85 78

The result has the subject names as a column index label (subject). reset_index and rename_axis clean this up if needed.

(long
 .pivot(index='name', columns='subject', values='score')
 .reset_index()
 .rename_axis(columns=None))
name english maths science
0 Alice 85 90 88
1 Bob 80 75 92
2 Charlie 70 85 78
